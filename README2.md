# 🎤 Early Speaker Recognition
> **"최소 몇 초의 음성이 주어졌을 때부터 화자를 신뢰할 수 있게 인식할 수 있는가?"**

음성의 초기 구간(0.3초 ~ 1.5초)만으로 화자를 식별하고, 시간 정보 활용 방식이 다른 5가지 모델의 duration별 성능을 비교한 시계열 분석 프로젝트입니다.

---

## 📌 프로젝트 목표

실시간 음성 보안 시스템과 같은 환경에서는 음성 전체를 기다릴 수 없습니다. 발화 자체가 짧거나, 소음 환경에서 유효 구간이 잘려나가거나, 시스템이 빠른 응답을 요구하는 경우가 많습니다.

음성은 시간축을 따라 변화하는 시계열 신호이고, 입력 길이가 짧아질수록 모델이 활용할 수 있는 시간적 패턴의 양이 줄어듭니다. 이러한 배경에서 다음 질문을 출발점으로 삼습니다.

> **"최소 몇 초의 음성이 주어졌을 때부터 화자를 신뢰할 수 있게 인식할 수 있는가?"**

이 질문에 답하기 위해 0.3초부터 1.5초까지 6단계 duration 구간을 설정하고, 시간 정보를 활용하는 방식이 서로 다른 5가지 모델을 비교 실험했습니다.

### 데이터 분할 방식

화자 단위가 아닌 **파일 단위**로 train/test를 분리했습니다. 각 화자의 발화 파일을 무작위로 섞어 80%는 train, 20%는 test로 나눴습니다. 이렇게 하면 test에도 train에서 학습한 모든 화자가 포함되어, "학습한 화자를 얼마나 잘 인식하는가(Closed-set)"를 정확하게 평가할 수 있습니다.

> **Closed-set Speaker Recognition**: 테스트 시 등장하는 화자가 반드시 학습 데이터에 포함되어 있다고 가정하는 설정. 본 실험의 목적은 "처음 보는 화자를 맞히는 것"이 아니라 "학습한 화자의 음성이 몇 초 이상이면 신뢰할 수 있게 인식되는가"이므로 이 설정이 적합합니다.

## 🔧 전처리 파이프라인

### 전체 흐름

```
원본 WAV + JSON
      ↓
① SpeechStart ~ SpeechEnd 구간 추출
      ↓
② RMS 음량 정규화 (target_rms=0.1)
      ↓
③ librosa.effects.trim (앞뒤 무음 제거, top_db=25)
      ↓
④ WebRTC VAD (발화 중간 침묵 제거, 30ms 프레임)
      ↓
⑤ Log-Mel Spectrogram 추출 → shape: (64, T)
      ↓
⑥ Duration별 슬라이싱 (패딩 없이 스킵 방식)
      ↓
⑦ 샘플별 정규화 (슬라이싱 후 적용)
      ↓
.npy 저장
```

### 단계별 설명

**① SpeechStart ~ SpeechEnd 구간 추출**

AI Hub 데이터셋의 JSON 파일에는 각 발화의 실제 음성 시작/종료 시점이 `SpeechStart`, `SpeechEnd`로 기록되어 있습니다. WAV 파일 전체가 아니라 이 구간만 잘라서 사용합니다. 파일 앞뒤의 불필요한 무음 구간을 제거해 이후 단계의 처리 효율을 높이기 위함입니다.

**② RMS 음량 정규화**

화자마다, 녹음 환경마다 음량 크기가 다릅니다. 이를 그대로 두면 모델이 화자의 목소리 특성이 아니라 음량 차이를 학습할 수 있습니다. RMS(Root Mean Square, 신호의 평균적인 음량 크기)를 기준으로 모든 발화의 음량을 동일한 수준(target RMS = 0.1)으로 맞춰 이 문제를 방지합니다.

> **RMS**: 각 샘플 값을 제곱한 후 전체 평균을 구하고 제곱근을 취한 값. "이 음성이 평균적으로 얼마나 큰 소리인가"를 하나의 숫자로 나타낸 지표.

**③ 에너지 기반 트리밍**

① 단계를 거쳤더라도 앞뒤에 작은 소음이나 무음이 남아 있을 수 있습니다. librosa의 `effects.trim`을 사용해 에너지가 낮은 앞뒤 구간을 추가로 제거합니다.

**④ VAD (Voice Activity Detection)**

발화 중간에 끼어 있는 침묵 구간을 제거합니다. 예를 들어 "안녕하세요 [잠깐 멈춤] 반갑습니다"처럼 중간에 쉬는 구간이 있으면, 그 침묵 프레임을 제거하고 실제 발화 프레임만 이어붙입니다. 구글이 개발한 실시간 통신 기술 WebRTC의 VAD 모듈을 사용했으며, 30ms 단위 프레임으로 음성/비음성을 판별합니다. VAD 후 유효 프레임이 하나도 없으면 원본을 그대로 유지합니다.

**⑤ Log-Mel Spectrogram 추출**

전처리된 wav 신호를 모델이 학습할 수 있는 2D 특징 맵으로 변환합니다.

- **Spectrogram**: 음성 신호를 "어떤 시점에 어떤 주파수가 얼마나 강한가"로 변환한 2D 이미지. x축이 시간, y축이 주파수, 색깔이 에너지 크기.
- **Mel**: 사람의 귀는 낮은 주파수 변화에 민감하고 높은 주파수 변화에 둔감합니다. Mel은 이런 청각 특성을 반영한 주파수 단위이며, Mel Spectrogram은 이 스케일로 주파수 축을 변환한 것입니다.
- **Log**: 소리 에너지의 범위가 너무 넓어서 그대로 쓰면 작은 차이가 묻혀버리므로, 로그를 취해 큰 값은 압축하고 작은 값은 상대적으로 부각되도록 합니다.

결과 shape은 `(64, T)`로, 64개의 Mel 필터 채널과 시간 프레임 T로 구성됩니다.

고정 파라미터는 다음과 같습니다. 팀원 전원이 동일한 값을 사용해 모델 간 비교 조건을 통일했습니다.

```python
SR          = 16000   # 샘플링 레이트
N_MELS      = 64      # Mel 필터 수
HOP_LENGTH  = 160     # 10ms (1초 = 100프레임)
N_FFT       = 400     # 25ms
FMIN        = 20
FMAX        = 8000
DURATIONS   = [0.3, 0.5, 0.7, 1.0, 1.3, 1.5]
MIN_DURATION = 0.3
RANDOM_SEED = 42
```

**⑥ Duration별 슬라이싱**

추출된 Log-Mel Spectrogram의 앞부분을 duration 길이만큼 잘라냅니다. 예를 들어 0.3초 duration이면 앞에서 30프레임(0.3s × 16000 / 160)만 사용합니다. 발화가 해당 duration보다 짧으면 **패딩 없이 스킵**합니다. 패딩을 하지 않는 이유는 패딩된 무음 구간이 모델 학습에 노이즈로 작용할 수 있기 때문입니다.

**⑦ 샘플별 정규화**

슬라이싱 후 각 샘플의 평균을 0, 표준편차를 1로 맞추는 표준화를 적용합니다. 이 단계는 반드시 슬라이싱 후에 수행해야 합니다. 슬라이싱 전에 정규화하면 잘려나간 구간의 통계값이 반영되어 실제 사용하는 구간의 분포가 달라질 수 있기 때문입니다.

```python
feat = (feat - feat.mean()) / (feat.std() + 1e-6)
```

### Duration별 사용 가능 발화 수

1.5초를 초과하면 사용 가능한 발화 수가 급격히 줄어들므로 실험 조건의 일관성을 위해 1.5초를 상한으로 설정하였습니다.

| 0.3s | 3,854 | 100.0% | 980 | 100.0% |
| 0.5s | 3,845 | 99.8% | 978 | 99.8% |
| 0.7s | 3,837 | 99.6% | 978 | 99.8% |
| 1.0s | 3,802 | 98.7% | 970 | 99.0% |
| 1.3s | 3,622 | 94.0% | 936 | 95.5% |
| 1.5s | 3,261 | 84.6% | 851 | 86.8% |

## 🤖 실험 설계

### 실험의 배경

팀원들이 각자 화자 인식 문제에 접근하면서 자유롭게 모델을 선택해 실험을 수행했고, 그 결과를 취합했을 때 LightGBM, 1D-CNN, TCN, ResNet34, x-vector, ECAPA-TDNN 총 6가지 모델이 사용되었습니다. 초기 실험에서 1.0초 부근이 성능 급등 변곡점인 것으로 관찰되었습니다.

이후 팀은 이 모델들을 공정하게 비교하기 위해 전처리 파이프라인과 파라미터를 통일하고, 동일한 데이터와 조건 하에서 duration별 성능을 측정하는 본격적인 비교 실험을 설계했습니다.

### 모델 구성 — 시간 정보 활용 정도에 따른 계단식 구조

취합된 모델들을 살펴보면 시간 정보를 활용하는 방식에 따라 자연스럽게 단계적 구조를 이룹니다.

| 단계 | 모델 | 시간 정보 활용 방식 |
|------|------|-------------------|
| 1단계 | LightGBM | 시간 정보 무시 — Mel Spectrogram을 평탄화해 통계적 특징만 사용 |
| 2단계 | 1D-CNN | 시간축 방향의 지역적 패턴 포착 |
| 3단계 | TCN | Dilated Convolution으로 장기 시계열 의존성 포착 |
| 4단계 | ResNet34 | Mel Spectrogram을 2D 이미지로 보고 시간-주파수 패턴 포착 |
| 5단계 | x-vector / ECAPA-TDNN | 발화 전체를 고정 길이 임베딩으로 압축, 실제 서비스 수준 |

이 구조를 통해 "시간 정보를 얼마나 잘 활용하느냐"에 따라 성능이 어떻게 달라지는지를 함께 분석합니다.

### 평가 지표

- **Accuracy**: 전체 예측 중 맞힌 비율. 직관적이지만 화자별 발화 수가 불균형할 경우 과대평가될 수 있습니다.
- **Macro F1-score**: 각 화자별 F1-score를 계산한 뒤 평균을 낸 값. 화자 수가 많고 발화 수 불균형이 있는 경우에도 모든 화자를 동등하게 평가합니다. **본 연구의 주요 지표입니다.**

> **F1-score**: Precision(모델이 "A다"라고 한 것 중 실제로 A인 비율)과 Recall(실제 A인 것 중 모델이 맞힌 비율)의 조화 평균. 둘 다 높아야 F1도 높아집니다.

### 학습 설정

| 항목 | 값 |
|------|-----|
| Optimizer | Adam (lr=1e-3) |
| Scheduler | CosineAnnealingLR |
| Epochs | 30 |
| Batch Size | 64 |
| Loss | CrossEntropyLoss |
| Device | GPU (Colab T4) |
| Seed | 42 |

duration마다 모델을 새로 초기화하고 seed를 고정해 비교 조건을 동일하게 유지했습니다.

## 📊 실험 결과

### 2단계: 1D-CNN

#### 모델 구조

1D-CNN은 Log-Mel Spectrogram의 시간축 방향으로 합성곱 연산을 수행해 지역적인 시간 패턴을 추출하는 모델입니다. "이 시점에서 저 시점으로 넘어가는 동안 어떤 소리 패턴이 반복되는가"를 학습하는 구조입니다. 사람의 목소리는 발음 습관, 음색, 말하는 속도 등 화자마다 고유한 시간적 패턴을 가지는데, 1D-CNN이 이 패턴을 포착합니다.

```
입력 (64, T)
    ↓
Conv1d(64→128, kernel=3) + BatchNorm + ReLU + MaxPool
: 64채널을 128채널로 확장하며 지역 패턴 추출, MaxPool로 크기 절반 축소
    ↓
Conv1d(128→256, kernel=3) + BatchNorm + ReLU + MaxPool
: 더 추상적인 고수준 패턴 추출, 다시 크기 절반 축소
    ↓
Conv1d(256→256, kernel=3) + BatchNorm + ReLU + AdaptiveAvgPool
: 마지막 합성곱 후 AdaptiveAvgPool로 시간 길이에 관계없이 256차원 벡터로 압축
    ↓
Linear(256→128) + ReLU + Dropout(0.3)
: 불필요한 특징 제거, 과적합 방지
    ↓
Linear(128→49)
: 49명 화자 중 누구인지 최종 분류
```

**AdaptiveAvgPool**이 핵심 역할을 합니다. duration마다 T(시간 프레임 수)가 다른데, 이 레이어 덕분에 0.3s든 1.5s든 동일한 모델 구조로 처리할 수 있습니다. T가 달라도 항상 256차원 벡터로 압축해주기 때문입니다.

#### 실험 결과

| Duration | Accuracy | Macro F1 |
|----------|----------|----------|
| 0.3s | 81.84% | 76.22% |
| 0.5s | 88.45% | 82.57% |
| 0.7s | 91.21% | 87.08% |
| 1.0s | 94.43% | 91.71% |
| 1.3s | 96.47% | 94.18% |
| 1.5s | 97.06% | 95.49% |

#### 결과 분석

duration이 길어질수록 성능이 꾸준히 상승하는 단조 증가 패턴을 보입니다. 0.3s에서 이미 Accuracy 81.8%를 달성하며, 0.7s부터 90%를 넘어섭니다.

성능 향상 폭을 보면 0.3s→0.5s 구간에서 Macro F1이 6.35%p로 가장 크게 오르고, 이후 구간마다 상승폭이 줄어들어 1.3s→1.5s 구간에서는 1.31%p에 그칩니다. duration이 길어질수록 추가되는 음성 정보의 효용이 점점 감소하는 수렴 패턴입니다.

**0.7초부터 유의미한 화자 인식이 가능했으며, 1.0초부터는 실용적으로 안정적인 성능 구간에 진입한 것으로 판단됩니다.** 전체 발화의 99.6%가 0.7초 이상이고, 이 시점에서 Accuracy 91%, Macro F1 87%를 달성해 충분한 인식 성능을 확보합니다. 1.0초 이후로는 성능 향상 폭이 급격히 줄어들어 더 긴 음성을 기다리는 것의 실용적 이점이 크지 않습니다.

### 3단계: TCN (Temporal Convolutional Network)

#### 모델 구조

TCN은 RNN(LSTM, GRU)이나 Transformer 없이 합성곱만으로 긴 시계열의 의존성을 포착하도록 설계된 모델입니다. 핵심 아이디어는 **Dilated Convolution**입니다.

일반 합성곱은 바로 인접한 프레임만 참조하지만, Dilated Convolution은 일정 간격을 두고 떨어진 프레임을 참조합니다. dilation을 1→2→4→8로 지수적으로 늘리면 층이 쌓일수록 참조하는 시간 범위가 기하급수적으로 넓어집니다.

```
dilation=1 : [■ ■ ■]                             → 인접 3프레임 참조
dilation=2 : [■ · ■ · ■]                         → 2칸 간격으로 5프레임 범위 참조
dilation=4 : [■ · · · ■ · · · ■]                 → 4칸 간격으로 9프레임 범위 참조
dilation=8 : [■ · · · · · · · ■ · · · · · · · ■] → 더 넓은 범위 참조
```

각 층은 **TemporalBlock**이라는 단위로 구성됩니다.

```
입력
    ↓
Dilated Conv → Chomp(인과성 유지) → ReLU → Dropout
    ↓
Dilated Conv → Chomp(인과성 유지) → ReLU → Dropout
    ↓
Residual Connection (입력을 그대로 더함)
    ↓
출력
```

- **Chomp**: 미래 시점의 정보가 현재 예측에 사용되지 않도록 잘라내는 역할. 현재까지의 음성만으로 판단해야 하는 상황에서 중요합니다.
- **Residual Connection**: 층이 깊어져도 학습이 잘 되도록 입력을 출력에 직접 더해주는 구조.

전체 처리 흐름은 다음과 같습니다.

```
입력 (64, T)
    ↓
TemporalBlock(dilation=1, 채널: 64→128)  : 인접 구간의 지역 패턴 포착
    ↓
TemporalBlock(dilation=2, 채널: 128→128) : 조금 더 넓은 범위 패턴 포착
    ↓
TemporalBlock(dilation=4, 채널: 128→256) : 중간 범위 시계열 의존성 포착
    ↓
TemporalBlock(dilation=8, 채널: 256→256) : 넓은 범위 시계열 의존성 포착
    ↓
AdaptiveAvgPool → 256차원 벡터로 압축
    ↓
Linear(256→128) + ReLU + Dropout(0.2)
    ↓
Linear(128→49) : 49명 화자 분류
```

학습 설정은 1D-CNN과 동일하게 맞춰 공정한 비교 조건을 유지했습니다.

#### 실험 결과

| Duration | Accuracy | Macro F1 |
|----------|----------|----------|
| 0.3s | 69.39% | 60.29% |
| 0.5s | 73.42% | 65.17% |
| 0.7s | 76.79% | 67.57% |
| 1.0s | 80.82% | 72.11% |
| 1.3s | 85.58% | 76.55% |
| 1.5s | 82.02% | 74.00% |

#### 결과 분석

1.3s까지는 duration이 길어질수록 꾸준히 성능이 오르다가, 1.5s에서 Accuracy 82.0%, Macro F1 74.0%로 오히려 하락하는 패턴이 나타났습니다. 이 현상이 학습 부족 때문인지 확인하기 위해 에폭을 30에서 50으로 늘려 재실험했으나 동일한 패턴이 유지되었습니다.

원인은 **receptive field 한계**로 판단됩니다. dilation이 1→2→4→8인 4층 구조에서 모델이 참조할 수 있는 시간 범위는 1.5s(150프레임) 전체를 커버하기에는 제한적입니다. 즉 입력이 길어져도 모델이 그 정보를 충분히 활용하지 못하는 구조적 한계가 있습니다.

만약 dilation을 1→2→4→8→16→32로 6층으로 확장하면 receptive field가 커져 이 문제를 완화할 수 있으나, 본 실험의 범위를 벗어나므로 향후 과제로 남깁니다.

### 1D-CNN vs TCN 비교
![비교.png]
![정확도.png]

| Duration | 1D-CNN F1 | TCN F1 | 격차 |
|----------|-----------|--------|------|
| 0.3s | 76.22% | 60.29% | 15.93%p |
| 0.5s | 82.57% | 65.17% | 17.40%p |
| 0.7s | 87.08% | 67.57% | 19.51%p |
| 1.0s | 91.71% | 72.11% | 19.60%p |
| 1.3s | 94.18% | 76.55% | 17.63%p |
| 1.5s | 95.49% | 74.00% | 21.49%p |

전 구간에서 1D-CNN이 TCN을 앞섰습니다.

TCN은 시간 흐름을 길게 보는 데 강점이 있는 모델입니다. 가까운 순간뿐 아니라 조금 떨어진 시점의 정보까지 함께 보면서 패턴을 학습합니다. 하지만 이번 실험에서는 이런 장점이 크게 활용되지 못한 것으로 보입니다.

화자 인식에서는 "무슨 내용을 말했는가"보다 "어떤 목소리로 말했는가"가 더 중요합니다. 즉 문장의 긴 흐름보다는 음색, 발음 습관, 목소리의 고유한 특징 같은 **짧은 구간의 지역적 패턴**이 더 중요한 문제였습니다. 1D-CNN은 이런 짧은 구간의 지역적 패턴을 잘 찾는 구조이기 때문에 전 구간에서 안정적으로 높은 성능을 보였습니다.

반면 TCN은 입력 길이가 길어질수록 더 넓은 시간 범위를 봐야 하는데, 현재 모델 구조로는 긴 음성 전체를 충분히 활용하지 못했을 가능성이 있습니다. 특히 1.5초 구간에서는 입력은 길어졌지만 실제로 모델이 효과적으로 사용하는 정보는 제한적이어서 성능이 오히려 감소한 것으로 보입니다. 또한 duration이 길어질수록 사용 가능한 학습 데이터 수가 줄어든 점도 영향을 주었을 가능성이 있습니다.

이는 1D-CNN이 TCN보다 전 구간에서 높은 성능을 보인 본 실험 결과로부터 도출한 해석이며, 화자 인식 태스크의 특성상 짧은 구간의 지역적 패턴이 더 결정적인 정보를 담고 있을 가능성을 시사합니다.

## 🔍 핵심 결론

> **0.7초부터 유의미한 화자 인식이 가능하며, 1.0초부터 실용적으로 안정적인 성능 구간에 진입한다.**

- **0.7s**: Accuracy 91%, Macro F1 87% 달성 (전체 발화의 99.6% 커버)
- **1.0s 이후**: 성능 향상 폭이 감소하는 수렴 패턴 진입
- **화자 인식 태스크**에서는 장기 의존성보다 단기 지역 패턴이 더 결정적임을 확인
- 단순하지만 지역 패턴에 특화된 1D-CNN이, 장기 의존성 모델인 TCN보다 전 구간에서 우세

이는 "긴 시계열이 항상 좋다"는 것이 아니라, **태스크마다 필요한 시계열 길이와 구조가 다르다**는 것을 실험적으로 보여주는 결과입니다.
